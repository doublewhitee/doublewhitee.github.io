1. 简介

几乎每种编程语言中，都有集合结构。集合比较常见的实现方式是哈希表。集合通常是由一组无序的、不能重复的元素构成。ES6中包含了Set类，可以直接使用。

2. 集合结构实现

2.1 常用方法

  • add(value)：向集合添加一个新的项；
  • remove(value)：从集合移除一个项；
  • has(value)：如果值在集合中，返回true，反之返回false；
  • clear()：移除集合中的所有项；
  • size()：返回集合所包含的元素的数量；
  • values()：返回一个包含集合中所有值的数组。

4. 哈希表

哈希表通常是基于数组进行实现的，但是相对于数组，它具有很多优势：（1）它可以提供非常快速的插入-删除-查找操作（2）无论多少数据，插入和删除需要接近常量的时间，即O(1)的时间级。实际上，只需要几个机器指令即可完成。（3）哈希表的速度比树还要快，基本可以瞬间查到想要的元素（4）哈希表相对于树来说编码要容易很多。

哈希表相对于数组的一些不足：（1）哈希表中的数据是没有顺序的，所以不能以一种固定的方式（例如从小到大）来遍历其中的元素。（2）通常情况下，哈希表中的key是不允许重复的，不能放置相同的key，用于保存不同的元素。

哈希表（散列表）是根据关键码值(Key value)而直接进行访问的数据结构。也就是说，它通过把关键码值映射到表中一个位置来访问记录，以加快查找的速度。

4.1 哈希化 & 冲突

哈希化：将大数字转化成数组范围内下标的过程。

哈希函数：通常我们会将单词转成大数字，大数字在进行哈希化的代码实现放在一个函数中，这个函数称为哈希函数。

哈希表：最终将数据插入到的这个数组，对整个结构的封装称之为一个哈希表。

冲突：如果两个关键字映射到了同一个位置，则称为哈希冲突（k1 != k2, f(k1) = f(k2)）。冲突不可避免，我们只能解决冲突。

解决冲突通常有两种方法：链地址法、开放地址法

4.1.1 链地址法

链地址法解决冲突的做法是：将所有关键字为同义词的结点链接在同一个单链表中。若选定的散列表长度为m，则可将散列表定义为一个由m个头指针组成的指针数组T[0..m-1]。凡是散列地址为 i 的结点，均插入到以T[i]为头指针的单链表中。

4.1.2 开放地址法

开放地址法的主要工作方式是寻找空白的位置来放置冲突的数据。但是探索这个位置的方式不同，有三种方法：（1）线性探测（2）二次探测（3）再哈希法。

线性探测：线性的查找空白的单元。注意在删除操作时不可以将这个位置的下表内容设置为null，因为可能会影响后续查询其他操作。所以通常删除一个位置的数据项时，可以将它进行特殊处理（比如设置为-1）。线性探测有一个比较严重的问题：聚集。聚集是一连串的填充单元，会影响哈希表的性能（插入、查询、删除），通过二次探测可以解决一部分这个问题。

二次探测：二次探测主要优化的是探测是的步长，一次性探测比较长的距离，避免聚集带来的影响。但是二次探测依然存在一些问题。例如连续插入步长相同数据，这种情况会造成步长不一的一种聚集。

再哈希法：我们需要一种依赖关键字的探测顺序，而不是每个关键字都一样。那么不同关键字即使映射到相同的数组下标，也可以使用不同的探测序列。再哈希化的做法为：把关键字用另外一个哈希函数再做一次哈希化，用这次哈希化的结果作为步长。对于指定的关键字，步长在整个探测序列中是不变的，不过不同的关键字使用不同的步长。

第二次哈希化需要具备以下特点：（1）和第一个哈希函数不同（2）不能输出为0（否则没有步长）

stepSize = constant - ( key % constant )

其中constant是质数，且小于数组的容量

4.1.3 哈希表的效率

哈希表中执行插入和搜索的效率是非常高的。如果没有发生冲突，效率就会更高；如果发生冲突，存取时间就依赖后来的探测长度。

平均探测长度以及平均存取时间取决于填装因子，随着填装因子变大，探测长度也越来越长。随着填装因子变大，效率下降的情况，在不同开放地址法中比链地址法更严重。

填装因子：表示当前哈希表中已经包含的数据项和整个哈希表长度的比值。

填装因子 = 总数据项 / 哈希表长度

开放地址法的装填因子最大为1，因为必须找到空白的单元才能将元素放入。

链地址法的装填因子可以大于1，因为拉链法可以无限的延伸下去。
